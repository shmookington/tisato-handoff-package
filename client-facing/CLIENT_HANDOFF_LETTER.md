# TISATO Website Ownership Handoff

TISATO Transportation Services now has a structured website handoff package containing the public site, launch materials, and technical blueprints required to operate the booking, admin, and email systems under the client's own hosting account.

This package is designed so the client or their chosen technical provider can move in phases instead of relying on a third-party personal hosting account.

## What Is Included

- A static public website package ready for standard web hosting.
- A booking form interface and request flow prototype.
- A static admin portal prototype showing how booking requests should be reviewed.
- Production-ready system documentation for booking intake, admin review, email notifications, and future SMS readiness.
- Hosting instructions for Netlify, Vercel, and traditional cPanel-style hosting.
- Launch checklist, environment variable checklist, and maintenance expectations.
- Recovered brand/reference notes and logo assets.

## What Must Be Connected Before Production

- A production database for ride requests.
- Secure admin authentication.
- Email provider credentials for dispatch and passenger notifications.
- Domain/DNS setup.
- Any optional SMS/A2P messaging provider.

## Recommended Launch Approach

The client should first publish the static public website, then connect the booking and admin systems as Phase 2. This keeps the public site online quickly while allowing the protected operational systems to be implemented carefully.

## Ownership Note

Once the client hosts this package under their own account, they control the domain, files, hosting provider, data, and future vendor relationships.
